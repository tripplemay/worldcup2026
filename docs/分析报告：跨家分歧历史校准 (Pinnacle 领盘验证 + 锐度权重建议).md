# 分析报告:跨家分歧 · 历史校准(Pinnacle 领盘验证 + 锐度权重建议)

**日期:** 2026-06-22
**脚本:** `scripts/divergence-calib.mjs`(纯离线、零 API 配额;trueIP3/分歧算法镜像 `lib/odds/{trueIP,bookDivergence}.ts`)
**数据:** football-data.co.uk,5 联赛(英超/西甲/德甲/意甲/法甲)× 3 季(2023-24/24-25/25-26),**n=4405 场**,含 6 家具名博彩的**开盘 + 收盘** 1X2 + 真实赛果。逐季可用家集:2324 = B365/BW/PS/WH/IW/VC,2425 = B365/BW/BF/PS/WH/1XB,2526 = B365/BW/PS。
**验证:** 3 个独立审计 agent(防泄漏/独立复算/微观结构怀疑论)+ 综合 agent(workflow `wf_af6087c6`)。

---

## TL;DR(经对抗验证后的诚实结论)

1. **真·领先指标 = Pinnacle(其次 B365)。** 逐家 leave-one-out lead-lag:**Pinnacle 开盘偏离其余家共识,能显著预测其余家的收盘移动**(leadScore≈**0.17**,跨 3 季稳定 0.21/0.15/0.14,且通过**安慰剂检验**——改用 Pinnacle *收盘*偏离则信号坍缩 0.17→0.035,证明信息在**早盘**而非同期回归)。复现业内公认「Pinnacle 领盘」结论 → **校准管线可信**。
2. **分歧"大小"几乎没有信息量。** 开盘 spread 与后续线动相关性 ≈ 0(干净的 TV 指标 r≈0.004)。**读盘价值不在"分歧多大",而在"谁在领跑"。**
3. **临盘市场已收敛、高度有效。** 各家收盘 Brier 几乎相同(~0.566–0.571,匹配样本内 PS vs 共识 z=−0.3 平手),收盘仅比开盘锐 0.0014(z=2,实务上可忽略)。Pinnacle 收盘居中**不矛盾**——它早盘领先,临盘大家都跟上了。
4. **可行动升级:用"领跑家身份"作先验,而非分歧幅度。** 生产 the-odds-api(regions=eu)**确实含 Pinnacle + Betfair Exchange + Matchbook**(共 25 家)→ 升级方向落地可行(见 §5)。

## 1. 方法

每场:对每家 1X2 用 `trueIP3` **逐家去水** → 共识 = 各路中位数(对软盘离群稳健,重归一化)。
- **lead-lag(H2b,核心):** 对每家 B,leave-one-out——`bDev = B开盘 − 其余家开盘共识`,`oMove = 其余家收盘共识 − 其余家开盘共识`;`leadScore = corr(bDev, oMove)`(跨场×3 路)。B 被完整排除在"其余家"之外 → **无泄漏**(审计确认)。
- **线动(H1):** `TV = ½·Σ|收盘共识 − 开盘共识|`(总变差,百分点)。
- **可靠性/锐度(H3/H4):** 多类 Brier `Σ(p−y)²`,按分歧分级 / 逐家 / 开盘vs收盘。

## 2. H2b 逐家领先排序(核心发现)

| 家 | leadScore | 判定 | 备注 |
|---|---|---|---|
| **PS = Pinnacle** | **0.171** | **真·领先**(3 季稳定 + 安慰剂通过) | n≈4401(3 季) |
| B365 = Bet365 | 0.088 | 次领先(z≈5) | n≈4401(3 季) |
| VC | 0.110 | 次领先 | 仅 2324 |
| BF = Betfair | 0.128 | **暂定**(降权) | **仅 2425 单季、且是 Betfair Sportsbook(overround~1.05)非 Exchange** |
| IW / 1XB | ≈0(+0.03/−0.02) | **噪声**(跨零,非滞后) | 小样本 |
| WH | −0.034 | 边际滞后(z≈−1.9) | — |
| **BW** | **−0.081** | **稳健滞后**(z≈−4.9,跨季稳定) | 软盘 |

> **效应量用 leadScore(≈0.17),不用 beta。** 早期口径里的「beta=0.58 = 市场朝 Pinnacle 走 58%」**已撤回**:beta 随对照家数变化(2 家 0.41 → ≥4 家 0.65),混入小基数均值回归,不稳健。leadScore 在所有口径下稳定,是该报的效应量。

## 3. H1 / H3 / H4(弱/混淆/收敛,均已软化)

- **H1 分歧→线动:** corr(spread, TV)=0.004 ≈ 0;corr(spread, |Δtop|)=0.057 但 |Δtop| 锚在**移动最小的平局侧**(审计:topSide 1.68pp < 任意主队侧 2.08pp),是劣指标。**结论:分歧幅度不预测线动。**
- **H3 分歧↔共识可靠性:** 按分级 tight 0.577 / moderate 0.463 Brier(分歧大反而更准),但**与盘面结构部分混淆**(绝对 pp 极差在中等概率盘峰值)。审计:混淆是**部分非全部**(同热门强度桶内 moderate 仍胜 tight +0.024~0.120)。**操作上不据此声称任何方向**,但"纯盘面假象"的措辞也略过。
- **H4 逐家收盘 Brier:** 全在 ~0.566–0.571 一窄带(匹配样本内更收敛 0.5664–0.5674)→ **临盘已收敛**。表中"IW 最锐"是 1 季 n=861 假象,具体排序无意义,**结论(收敛/共识≈最锐单家)成立**。

## 4. 验证发现(均不推翻头条;已吸收进上文)

- ✅ leave-one-out **无泄漏**;PS 排序在每种口径、逐季、安慰剂下复现。
- ⚠️ **3 路 pooling 夸大所有 z/SE**(~13200 个线性相关行,有效 n≈4401 场)——**点估计/排序稳健,精度声明已下调**。
- ⚠️ **football-data "开盘"是周五/周二定时快照非真开盘** → lead-lag 测的是**压缩窗口**内的领先(真实 TV~2.5pp,测试有效)。
- ⚠️ CSV `split(',')` 对本数据集**安全**(0 错行),仅在移植到带引号源时有隐患。
- ⚠️ 脚本未做 lib 的 round4/spreadPp 1dp 取整 → 0.7% 场在阈值边界翻级(影响微)。

## 5. 生产可行性 + 建议

**生产 the-odds-api(regions=eu)25 家含锐盘:`pinnacle` ✅ / `betfair_ex_eu`(Betfair Exchange)✅ / `matchbook`(交易所)✅**,以及 marathonbet / 1xBet(高限额);其余多为休闲软盘(Unibet/WilliamHill/Betsson/Tipico/888/LeoVegas/Winamax/Betclic…)。

**✅ 建议升级(待用户定):用"领跑家身份/锐度"作先验,替代纯幅度读盘**
- 给已知锐盘打标(`pinnacle`/`betfair_ex_eu`/`matchbook`,可含 marathonbet/onexbet 为半锐),展示时把"领跑 = 锐盘"标为**真信号**(如「⚡锐盘领跑 Pinnacle 68%」),"领跑 = 软盘"则弱化为噪声。
- 更强变体:把头条共识改为**锐盘子集共识**,并展示**锐盘 vs 软市场**的差(这才是真正的 RLM/读盘信号)。
- 这与已上线的"跨家分歧"是**同一数据、纯展示层增量**(零额外配额),且**身份排序可移植**(Pinnacle 就是 Pinnacle)。

**❌ 不要做(校准明确证伪):**
- 不要移植 tight/moderate/wide **阈值**或任何 **beta 系数**——它们是 3–6 家面板的数据集特定产物,在 25 家生产feed下会失真(家数越多极差机械变大)。
- 不要把"分歧幅度"当线动/可靠性预测器(H1/H3 基本无信号)。
- 仅 SIGN/ORDER 可移植,系数不可。

## 6. 局限

- football-data 仅 3–6 家 vs 生产 ~25 家 → **幅度/阈值不可移植**(身份排序可)。
- "开盘"非真开盘 → 领先效应可能被压缩低估。
- 软盘排序(除 BW 稳健滞后外)多在噪声带,且未在 25 家生产面板上重测——**视为合理先验,非已证结论**。
- the-odds-api 在部分 US/UK region 可能不含 Pinnacle(本项目 eu region 含);若换 region 需复查锐盘覆盖。

---

**一句话:** 我们上线的跨家分歧,真正值钱的是**"领跑家是谁"**(Pinnacle/Bet365 领跑=信号,软盘领跑=噪声),而非"分歧多大";生产含 Pinnacle/Betfair/Matchbook,可据此做"锐度加权"的纯展示层升级,但切勿移植阈值与系数。
